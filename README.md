# &lt;TG /&gt; — Personal Portfolio

Live at **[tanmaygargav.github.io/portfolio](https://tanmaygargav.github.io/portfolio)**

Personal portfolio for Tanmay Gargav — Technical Architect & Team Lead at 47Billion. Covers technical skills, fitness (Strava-powered), photography, and contact.

---

## Stack

| Layer | Tech |
|---|---|
| Hosting | GitHub Pages (free, public repo) |
| Frontend | Single-file HTML + vanilla JS + CSS |
| Charts | Chart.js 4.4 (bar + donut) |
| Data | JSON files fetched at runtime via `fetch()` |
| Fitness data | Strava bulk CSV export → `sync.py` → `data/strava.json` |
| Content data | Google Sheets (published CSV) → `sync.py` → `data/*.json` |
| Config | `config.json` — all site settings, zero hardcoded content |

## Updating Fitness Data (Strava)

Strava's API requires a paid subscription, so this site uses bulk CSV export instead.

**Steps to sync:**

1. Go to Strava Settings → My Account → Download or Delete Your Account → Request Your Archive
2. Extract `activities.csv` from the downloaded zip
3. Place it at `strava_export/activities.csv`
4. Run the sync script:

```bash
# Normal sync
python3 sync.py

# Preview without writing files
python3 sync.py --dry-run

# Point to a custom CSV path
python3 sync.py --csv /path/to/activities.csv
```

5. Commit and push the updated data file:

```bash
git add data/strava.json
git commit -m "content: sync strava data YYYY-MM-DD"
git push
```

The `strava_export/` folder is gitignored — raw CSV files are never committed.

## Updating Content (Google Sheets)

1. Create a Google Sheet with tabs: `Skills`, `Content`, `Photos`
2. Publish it: File → Share → Publish to web → CSV
3. Copy the Sheet ID from the URL
4. Add to `.env`:

```
GOOGLE_SHEETS_ID=your_sheet_id_here
```

5. Run `python3 sync.py` — it will fetch each tab and write to `data/`.

## Local Development

No build step. Open `index.html` directly in a browser, or serve it locally to avoid CORS issues with `fetch()`:

```bash
# Python
python3 -m http.server 8080

# Node
npx serve .
```

Then open `http://localhost:8080`.

## Git Conventions

### Branch naming

| Type | Pattern | Example |
|---|---|---|
| New feature | `feature/<desc>` | `feature/fitness-pie-chart` |
| Bug fix | `fix/<desc>` | `fix/strava-stats-display` |
| Content update | `content/<desc>` | `content/update-skills` |
| Style / UI | `style/<desc>` | `style/social-icons` |
| Chore / tooling | `chore/<desc>` | `chore/strava-csv-sync` |

### Commit messages (Conventional Commits)

```
feat: add activity distribution donut chart
fix: use Gmail icon for @gmail.com addresses
style: brand SVG icons for all social links
content: sync strava data 2026-07-23
chore: update sync.py to support CSV export
```

### Workflow

```bash
git checkout -b feature/your-feature
# make changes
git add .
git commit -m "feat: your change"
git push --set-upstream origin feature/your-feature
# open PR → review → merge into main
```

`main` is protected — no direct pushes. All changes go through PRs.
